NC algorithms, live

The parallel slack and the serial fraction, side by side. Three NC algorithms run on your GPU through WebGPU — with the round count (the depth T, the quantity NC is about) displayed next to the wall clock — each verified exactly against a sequential baseline compiled to WebAssembly. And one P-complete problem that pointedly has no GPU column. The WGSL kernels for scan and bitonic sort come verbatim from the amdahl package’s bsd tier.

What this page is demonstrating

1 NC, and why rounds are the number that matters

NC (“Nick’s Class”, named by Cook for Nick Pippenger’s 1979 characterization [3]; taxonomy in Cook 1985 [4]) is the class of problems solvable in polylogarithmic depth with polynomially many processors. Depth — the length of the longest chain of dependent operations — is what survives hardware generations; Brent’s theorem [5] says work can always be respread over whatever processors exist, but depth cannot. Each card therefore prints its round count next to the milliseconds: 3 launches for the blocked scan, log² comparator rounds for Batcher, ⌈log₂n⌉ jumps for Wyllie. On a GPU there is no grid-wide barrier inside a dispatch, so the round is the kernel launch — the PRAM step made physical, which is the standing model of the amdahl package these kernels come from.

2 The serial baselines, and small-n honesty

Each GPU result is verified exactly (u32, mod 2³²) against a sequential implementation compiled from freestanding C to WebAssembly — and the baseline is the strongest natural serial competitor, not a strawman: one-pass accumulation for scan, LSD radix for sorting. At these sizes the tuned serial code sometimes wins the wall clock; that is not a bug in the theory, it is the theory. Amdahl’s 1967 argument [1] caps speedup by the serial fraction, dispatch overhead is a real serial fraction, and the asymptotic separation only pays past a crossover. Watching the crossover move as n grows is the point of the size selector.

3 The P-complete counterpoint

Horn unit resolution is P-complete (Jones–Laaser 1977 [8]; entry A.6.2 in Greenlaw–Hoover–Ruzzo [2]): every polynomial-time problem NC-reduces to it, so a polylog-round GPU kernel for it would prove NC = P. The card runs a chain-deep instance where the propagation depth equals the atom count — forward chaining that cannot be flattened, as far as forty years of effort can tell. It is the same algorithm as this site’s Horn clause solver, shaped adversarially. The empty GPU cell is the exhibit: this is the flat line the amdahl package is named for. Notably, the frontier moves: bipartite perfect matching sat in the open-problems appendix of [2] for three decades before the 2026 proof that it is in NC [9].
