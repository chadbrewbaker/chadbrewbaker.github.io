Shell syntax as railroad diagrams

The grammar you half-remember mid-pipe, drawn as tracks: bash pipelines and lists, redirections, ${parameter} expansions, and the synopses and expression grammars of find and grep — laid out automatically to a target width you control with the slider, after the diagram language and wrapping approach of Chiplunkar & Pit-Claudel’s Automatic layout of railroad diagrams [1]. Every diagram below is a small editable program in their notation; change it and the tracks re-lay themselves.

What this is

1 The diagram language

Diagrams here are terms in the little language of [1], Fig. 6: "label" is a terminal token, [label] a nonterminal, (d …) a sequence, and (+ d d) / (- d d) binary stacks — positive for alternation (top subdiagram on the main line, so (+ () d) is “optional d”), negative for loops (bottom subdiagram rides the return path, so (- d ",") is a comma-separated list and (- () d) is “zero or more”). Regular expressions and BNF translate into this language mechanically (their Table 1), which is how the example diagrams were written from the bash(1), zshexpn(1), find(1), and grep(1) manuals. The last example is the diagram language drawn in itself.

2 Layout, and what the width slider demonstrates

The paper’s central observation is that railroad layout is a 1.5-dimensional problem: like text, diagrams follow reading order and must wrap to a target width; unlike text, a row can fork, stack, and run backwards. Their compiler proceeds in three steps — vertical alignment, wrapping, horizontal justification — with wrapping framed as preference-ordered optimization. The engine on this page is an honest, small subset of that design: same diagram language, greedy first-fit wrapping toward the target width, default alignment everywhere, and nested positive stacks flattened into multi-way branches for rendering (their Fig. 7b). For the full treatment — principled wrap penalties, tip specifications, justification policies — see their implementation, librrd, with a web UI at systemf.epfl.ch/etc/librrd. Drag the slider and watch sequences re-wrap; the status line reports when a diagram has hit its min-content width and can shrink no further.

3 Why shell syntax is the right demo

Railroad diagrams began as documentation for programming languages — Wirth’s Pascal materials are the earliest published instance anyone has found [2] — and SQLite has famously documented its SQL dialect with them since 2008 [1, §4.2]. Shell grammar is a perfect match for the notation and a notorious mismatch for prose: the optional prefixes of a pipeline, the eight-way redirection operator choice, separator alternatives between list items, and find’s genuinely recursive expression grammar are all one glance here and three paragraphs in the manual. The diagrams are curated subsets, faithful to the cited manual sections but not exhaustive — bash(1) alone would need a book of tracks.
